# PetVerse API 认证和权限系统说明

## 系统概述

本系统实现了基于JWT的认证和基于UserTag的权限控制机制。

## 核心功能

### 1. JWT认证
- 使用HS256算法生成和验证JWT令牌
- 令牌包含用户ID、用户名、角色和标签信息
- 默认有效期2小时

### 2. 权限控制
- **普通用户**: 无tag或非"admin" tag的用户
- **管理员**: 拥有"admin" tag的用户

### 3. 白名单机制
以下接口无需认证即可访问：
- `/api/user/login` - 用户登录
- `/api/user/check-username/*` - 用户名检查
- `/api/user/check-email/*` - 邮箱检查
- `/health` - 健康检查
- `/swagger/*` - API文档

## 配置说明

### appsettings.json 配置项

```json
{
  "JwtSettings": {
    "SecretKey": "your-super-secret-key-that-should-be-at-least-32-characters-long-for-security",
    "Issuer": "PetVerse.Api",
    "Audience": "PetVerse.Client",
    "ExpirationMinutes": 120
  },
  "Auth": {
    "EnableGlobalAuthentication": true,
    "WhitelistRoutes": [
      "/api/user/login",
      "/api/user/check-username/",
      "/api/user/check-email/",
      "/health",
      "/swagger/*"
    ]
  }
}
```

## 使用方法

### 1. 用户登录获取令牌
```bash
POST /api/user/login
Content-Type: application/json

{
  "usernameOrEmail": "用户名或邮箱",
  "password": "密码"
}
```

响应包含JWT令牌：
```json
{
  "success": true,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expiresAt": "2026-02-14T17:00:00Z",
    "user": {
      "id": 1,
      "username": "testuser",
      "role": "User",
      "tags": ["vip", "premium"]
    }
  }
}
```

### 2. 使用令牌访问受保护接口
在请求头中添加Authorization:
```
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

### 3. 权限控制使用
在控制器或方法上添加Authorize属性：
```csharp
[Authorize(RequiredRole = UserRole.Admin)]
public async Task<IActionResult> AdminOnlyMethod()
{
    // 只有管理员可以访问
}
```

## 错误响应

### 401 Unauthorized
- 未提供认证令牌
- 令牌无效或已过期

### 403 Forbidden
- 权限不足，无法访问特定资源

## 安全建议

1. **密钥管理**: 在生产环境中使用强密钥并妥善保管
2. **HTTPS**: 强制使用HTTPS传输令牌
3. **令牌刷新**: 实现刷新令牌机制延长会话
4. **令牌撤销**: 考虑实现令牌黑名单功能
5. **速率限制**: 对登录接口实施速率限制防止暴力破解

- 完整的认证流程测试

## 注意事项

1. 系统默认启用全局认证，可通过配置关闭
2. UserTag表需要预先配置"admin"标签用于管理员识别
3. 权限检查基于用户拥有的标签进行判断
4. 所有认证失败都会返回统一格式的错误响应
